### Differentiate between Formal-Technical Review and Walkthrough Reviews

| Aspect              | FTR              | Walkthrough        |
| :------------------ | :--------------- | :----------------- |
| **Formality Level** | High             | Low                |
| **Main Goal**       | Find defects     | Gain understanding |
| **Leader**          | Moderator        | Author             |
| **Preparation**     | Mandatory        | Optional           |
| **Focus**           | Product          | Presenter logic    |
| **Data Recording**  | Formal log       | Informal notes     |
| **Attendees**       | Peers/Experts    | Any stakeholders   |
| **Output**          | Defect report    | General feedback   |
| **Follow-up**       | Strict & tracked | Informal           |
| **Structure**       | Rigid            | Flexible           |

### Differentiate between Alpha and Beta Testing

| Aspect               | Alpha Testing  | Beta Testing    |
| :------------------- | :------------- | :-------------- |
| **Location**      | Internal lab   | External site   |
| **Testers**       | Internal staff | Real users      |
| **Goal**          | Find bugs      | User feedback   |
| **Visibility**    | Private        | Public          |
| **Stability**     | Less stable    | More stable     |
| **Control**       | High control   | Low control     |
| **Feedback Type** | Technical      | User experience |
| **Phase**         | Before beta    | Before release  |
| **Data**          | Crash logs     | Usability       |
| **Duration**     | Long cycles    | Short cycles    |

---

### Differentiate between Scrum and Kanban

| Aspect            | Scrum             | Kanban             |
| :---------------- | :---------------- | :----------------- |
| **Cycles**     | Fixed sprints     | Continuous flow    |
| **Roles**      | Defined roles     | No fixed roles     |
| **Metrics**    | Velocity          | Lead time          |
| **Reset**      | Reset each sprint | Persistent         |
| **Commitment** | Sprint backlog    | WIP limits         |
| **Change**     | No mid-sprint     | Anytime            |
| **Events**     | Regular events    | No required events |
| **Artifacts**  | Specific set      | Minimal            |
| **Planning**   | Sprint planning   | On-demand          |
| **Focus**     | Iterative goals   | Flow efficiency    |

---

### Differentiate between White box and Black box Testing

| Aspect                    | White Box Testing          | Black Box Testing             |
| :------------------------ | :------------------------- | :---------------------------- |
| **Internal Knowledge** | Required                   | Not Required                  |
| **Tester Perspective** | Developer View             | User View                     |
| **Testing Basis**      | Code & Internal Structure  | Specifications & Requirements |
| **Skill Requirement**  | Programming Skills Needed  | No Programming Skills         |
| **Granularity**        | Fine-grained (Low Level)   | Coarse-grained (High Level)   |
| **Main Goal**          | Improve Code Quality       | Validate Functionality        |
| **Test Design**        | Based on Code Paths        | Based on Input/Output         |
| **Coverage Type**      | Code Coverage              | Requirement Coverage          |
| **Performed By**       | Developers & SDETs         | QA Testers                    |
| **Examples**          | Unit Testing, Path Testing | System Testing, UAT           |

### Differentiate between Agile and Evolutionary Models

| Factor | Agile Model | Evo Model |
| :--- | :--- | :--- |
| **Primary Focus** | Customer Satisfaction | Measurable Value |
| **Core Mechanism** | Iterative Sprints | Incremental Delivery |
| **Feedback Source** | Customer Collaboration | Stakeholder Feedback |
| **Change Handling** | Embracing Change | Requirement Prioritization |
| **Success Metric** | Working Software | Value Verification |
| **Team Structure** | Self-Organizing Teams | Metric-Driven |
| **Progress Pace** | Sustainable Development | Cost Measurement |
| **Delivery Approach** | Continuous Delivery | Evolutionary Design |
| **Communication Style** | Face-to-Face | Quantitative Feedback |
| **Value Tracking** | Business Value | Track Value |
